# Snake AI - Algorithm Discovery (SF231)

## 1. ปัญหาที่โปรแกรมนี้แก้ (Problem Statement)
โปรแกรมนี้แก้ปัญหาการหาเส้นทางที่สั้นที่สุด (Shortest Path) จากหัวงูไปยังอาหาร โดยต้องไม่ชนสิ่งกีดขวาง (ผนัง) และไม่ชนตัวเอง (ลำตัว) รวมถึงการวางแผนเพื่อความอยู่รอดในกรณีที่ไม่มีเส้นทางที่ปลอดภัยไปยังอาหาร

## 2. ขั้นตอนวิธี และโครงสร้างข้อมูล (Algorithms & Data Structures)

### ก. A* Search Algorithm
ใช้สำหรับการหาเส้นทางที่สั้นที่สุดไปยังอาหาร
- **เหตุผลที่เลือก:** A* มีประสิทธิภาพสูงกว่า BFS ในการหาเส้นทางบน Grid เพราะใช้ Heuristic มาช่วยลดจำนวน Node ที่ต้องตรวจสอบ
- **Heuristic:** ใช้ **Manhattan Distance** ($|x1 - x2| + |y1 - y2|$) เนื่องจากงูเคลื่อนที่ได้ 4 ทิศทาง (ข้ามทแยงไม่ได้)

### ข. Survival Heuristic (Flood Fill/BFS)
ใช้เมื่อ A* หาทางไปหาอาหารไม่เจอ หรือประเมินแล้วว่าทางนั้นอันตราย
- **เหตุผลที่เลือก:** ป้องกันไม่ให้งูเดินเข้าทางตัน (Dead end) โดยการคำนวณพื้นที่ว่างที่เหลือ (Reachable Space) และเลือกทางที่มีพื้นที่มากที่สุด

### ค. โครงสร้างข้อมูล (Data Structures)
1.  **Array/List:** ใช้เก็บส่วนต่างๆ ของลำตัวงู (Snake Body) เพื่อให้ง่ายต่อการเพิ่มหัว (unshift) และตัดหาง (pop)
2.  **Map/Set:** ใน A* ใช้เก็บ `gScore`, `fScore` และ `visited nodes` เพื่อให้เข้าถึงข้อมูลด้วย Key (พิกัด x,y) ได้ในเวลา $O(1)$
3.  **Queue:** ใช้ใน Flood Fill สำหรับ Survival Heuristic

## 3. วิเคราะห์ Big O (Big O Analysis)

| Operation | Algorithm | Big O (Worst Case) | คำอธิบาย |
| :--- | :--- | :--- | :--- |
| **Pathfinding** | A* Search | $O(N \log N)$ หรือ $O(N)$ | $N$ คือจำนวนช่องใน Grid (400 ช่อง) การใช้ Priority Queue จะเป็น $O(N \log N)$ แต่ในโปรแกรมนี้ใช้ Array search เป็น $O(N)$ |
| **Collision Check** | Linear Scan | $O(L)$ | $L$ คือความยาวของงู ต้องตรวจสอบทุกส่วนของลำตัว |
| **Survival Check** | Flood Fill | $O(N)$ | ตรวจสอบพื้นที่ว่างที่เชื่อมต่อกันทั้งหมดใน Grid |

## 4. วิจารณ์โค้ด AI (AI Critique)

### จุดแข็ง (Strengths)
- การแยก Logic ระหว่าง Game Engine และ AI ทำให้โค้ดอ่านง่ายและ Maintain ง่าย
- มีระบบ Fallback (Survival Heuristic) ทำให้งูไม่ตายง่ายๆ เมื่อไม่มีทางไปหาอาหาร
- การเลือกใช้ Manhattan Distance เหมาะสมกับข้อจำกัดของการเคลื่อนที่บน Grid

### จุดอ่อน (Weaknesses)
- **Lack of Long-term Planning:** A* สนใจแค่ทางที่สั้นที่สุดไปหาอาหาร แต่อาจจะทำให้งู "ขังตัวเอง" (Self-trapping) ได้ถ้าพื้นที่หลังกินอาหารแคบเกินไป
- **Performance:** การใช้ Flood Fill ทุกครั้งที่หาทางไม่เจออาจจะกิน Performance ใน Grid ขนาดใหญ่ (แต่ 20x20 ไม่มีปัญหา)

### สิ่งที่ปรับปรุงได้ (Improvements)
- **Hamiltonian Cycle:** หากต้องการให้งูไม่มีวันตาย (แต่อาจเดินช้าลง) สามารถใช้ Hamiltonian Cycle เพื่อเดินให้ครบทุกช่อง
- **Tail Following:** เพิ่ม Logic ให้งูเดินตามหางตัวเองเมื่อไม่มีทางไปหาอาหาร เพราะหางคือจุดที่พื้นที่กำลังจะว่างลงเสมอ
